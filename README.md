## 1. Difference Between Interface and Type Alias in TypeScript

টাইপস্ক্রিপ্টে `interface` এবং `type alias` উভয়ই ডাটা বা অবজেক্টের গঠন নির্ধারণ করতে ব্যবহৃত হয়। যদিও অনেক ক্ষেত্রে তারা একইভাবে ব্যবহার করা যায়, তারপরও তাদের মাঝে কিছু গুরুত্বপূর্ণ পার্থক্য রয়েছে।

---

## Interface

- প্রধানত অবজেক্ট বা ক্লাসের স্ট্রাকচার নির্ধারণের জন্য ব্যবহৃত হয়
- একই নামে একাধিকবার ঘোষণা করলে TypeScript এগুলোকে merge করে
- সহজে extend করা যায় এবং inheritance করা যায়
- এটা মডেলের স্ট্রাকচার নির্ধারণের জন্য বেশি উপযোগী

---

## Type Alias

- এটা আরও বহুমুখী; শুধু অবজেক্ট নয়, যেকোনো ধরনের টাইপ (primitive, union, tuple, function ইত্যাদি) নির্ধারণ করতে পারে
- একবার তৈরি হলে একই নামে পুনরায় তৈরি করা যায় না, অর্থাৎ merge হয় না
- intersection অথবা union ব্যবহার করে extend করা সম্ভব

- টাইপ ডিজাইন বা type combination এর ক্ষেত্রে বেশি কার্যকর

---

## 2. What is the use of the keyof keyword in TypeScript?

`keyof` হলো TypeScript এর একটি অপারেটর, যা কোনো অবজেক্ট টাইপের সব property-এর নামকে একটি union টাইপ হিসেবে প্রদান করে। এর মাধ্যমে টাইপ সেফ প্রপার্টি অ্যাক্সেস নিশ্চিত করা যায় এবং ভুল key ব্যবহার করা থেকে রক্ষা পাওয়া যায়।

---

### কেন ব্যবহার করা হয়?

- অবজেক্টের key-গুলোকে টাইপ হিসেবে পাওয়ার জন্য
- ভুল key অ্যাক্সেস থেকে রক্ষা পাওয়ার জন্য
- টাইপ সেফ ডাইনামিক প্রপার্টি অ্যাক্সেস করার জন্য
- Generic function-এ সঠিক key নিশ্চিত করার জন্য

---

### উদাহরণ

```ts
type User = {
  id: number;
  name: string;
  email: string;
};

type UserKeys = keyof User;
// Output: "id" | "name" | "email"
```

## 3. Difference Between `any`, `unknown`, and `never` in TypeScript

TypeScript এ `any`, `unknown`, এবং `never` — এই তিনটি টাইপ ব্যবহৃত হয় ভিন্ন ভিন্ন পরিস্থিতিতে। এরা কোডের type safety এবং error checking-এ সহায়তা করে

---

## `any`

- যে কোনো টাইপের ভ্যালু রাখা যায়
- এর ফলে টাইপ সেফটির সুবিধা হারিয়ে যায়
- যখন একজন ডেভেলপার নিশ্চিত নয় যে কোন টাইপ আসবে, তখন ব্যবহার করা হয়

> **সংক্ষেপে:** `any` → "আমাকে যা দিবা আমি তাই নিব"

---

## `unknown`

- যে কোনো ভ্যালু রাখা যায়, কিন্তু ব্যবহার করার আগে টাইপ চেক করতে হয়
- `any` থেকে বেশী সেইফ কারণ টাইপ না জেনে প্রোপার্টি অ্যাক্সেস করতে দেয় না
- ইনপুট ডাটা বা API response handling এর ক্ষেত্রে কাজে আসে

> **সংক্ষেপে:** `unknown` → “আমার কাছে ডাটা আছে , কিন্তু আগে টাইপ চেক করে নিতে হবে”

---

## `never`

- এমন একটি টাইপ যেটির **কখনো কোনো ভ্যালু থাকতে পারে না**
- সাধারণত এরর থ্রো করা ফাংশন, বা infinite loop এর ক্ষেত্রে আমরা এটি ব্যবহার করি
- Unreachable code বোঝাতে ব্যবহৃত হয়

> **সংক্ষেপে:** `never` → “আমার কাছে কোন কিছুই থাকতে পারবে না”

---
